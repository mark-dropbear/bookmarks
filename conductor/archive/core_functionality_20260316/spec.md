# Specification: Core Bookmarking Functionality

## Overview
Implement the foundational features of the local-first bookmark manager, including saving, viewing, and searching bookmarks using an in-memory repository and Material 3 UI components.

## Requirements
- Setup development environment with `@web/dev-server` and Import Maps managed via `jspm` CLI.
- Implement a Clean Architecture structure (Entities, Use Cases, Repositories, Adapters).
- Create a `Bookmark` entity and its corresponding in-memory repository.
- Build a Material 3 UI with Lit components:
    - Base Layout with routing via `@lit-labs/router`.
    - Data loading via `@lit/task`.
    - Dependency injection and state sharing via `@lit/context`.
    - `AddBookmark` form.
    - `BookmarkList` display.
    - `Search` filter.
- Support basic tagging for bookmarks.
- Ensure all code is documented with JSDoc.

## Technical Details
- **Storage:** In-memory repository (for initial phase, later IndexedDB).
- **UI:** Lit, `@material/web`, `@lit-labs/router`, `@lit/context`, `@lit/task`.
- **Loading:** ESM with Import Maps generated by JSPM CLI and `es-module-shims`.
- **Server:** `@web/dev-server`.
- **Testing:** `@web/test-runner` `@esm-bundle/chai` & `@web/test-runner-playwright`
